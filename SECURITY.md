# Security Policy

## Supported versions

Agent Team Orchestrator is pre-1.0. Security fixes are handled on the main development line unless a maintained release branch is announced.

## Reporting a vulnerability

Do not include vulnerability details, secrets, exploit payloads, or sensitive logs in a public issue.

If GitHub private vulnerability reporting is enabled for this repository, use it. If it is not enabled, open a minimal public issue asking the maintainers how to share details privately, without including sensitive details.

Maintainers will acknowledge actionable reports as availability permits and will coordinate fixes through normal project releases or commits.
